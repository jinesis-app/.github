# Jinesis

**Jinesis** is a platform designed to help **Catholic churches and ministries easily create and manage their digital presence**.
Our goal is to provide churches with a unified system for publishing information, managing schedules, sharing articles, and communicating with their communities online.

The platform allows each church to have its own website while maintaining a **consistent structure and shared infrastructure**.

---

# Mission

To provide churches and ministries with **modern, accessible, and reliable digital tools** that support evangelization, communication, and community engagement.

---

# Projects

The Jinesis organization currently maintains the following core projects.

## Jinesis API

The **Jinesis API** is the backend service that powers the platform.

Responsibilities include:

* Authentication and authorization
* Church management
* Content management
* Email delivery
* Address management
* API access for web and mobile applications
* Standardized API responses (HAL format)

Technologies:

* PHP
* Symfony
* Doctrine ORM
* REST API
* HAL (Hypertext Application Language)

---

## Jinesis Web

The **Jinesis Web** project is the main web application used by administrators and users.

Features include:

* Church website management
* Dashboard and administrative tools
* Article and schedule publishing
* Newsletter management
* Integration with the Jinesis API

Technologies:

* Vue.js
* Modern frontend tooling
* REST API integration

---

# Architecture Principles

Jinesis follows several architectural principles:

* **Domain-Driven Design (DDD)**
* **Modular architecture**
* **API-first development**
* **Clear separation between frontend and backend**
* **Scalable infrastructure**

These practices help ensure the platform remains maintainable as it grows.
